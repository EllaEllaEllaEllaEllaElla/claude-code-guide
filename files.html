Your First Files

Two files every project should have.

1

CLAUDE.md — Claude's sticky note

  1. Open any project folder in VS Code
  2. Click the ⚡ spark icon to open Claude and type:
create a CLAUDE.md file that describes this project
  1. Claude will write the file automatically — you don't need to touch it
  2. Do this at the start of every new project
Claude reads CLAUDE.md every time you open the project. It's how it remembers what you're building, what style you like, and what not to change. This is the #1 thing beginners skip — don't skip it.
2

progress.md — your learning log

  1. Inside projects, create a folder called ai-progress and open it in VS Code
  2. Click ⚡ and type:
create a file called progress.md where I can track what I've learned
  1. Add your first entry: what you set up today
A .md file is called markdown — it's plain text with light formatting. # makes a heading, - makes a bullet. That's really all you need to know.

You know the system.

Open folder → make CLAUDE.md → build. Every time.
