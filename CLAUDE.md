# Tech Learning Lab

This folder is Ella's personal space to learn, experiment, and build with Claude Code.

## About You
- Beginner CS student, learning by doing
- New to most tooling — assume nothing is obvious, explain the "why" not just the "what"
- Prefer short, clear explanations over jargon-heavy ones
- Okay to be corrected; prefer honest feedback over encouragement that glosses over problems

## How We Work Here
- This is a low-stakes sandbox. Mistakes are fine and expected.
- Break things down into small, testable steps.
- When introducing a new concept, explain it in plain English first, then show the code.
- Prefer runnable examples over abstract explanations.
- Flag when something is "real-world best practice" vs. "simplified for learning."

## Ground Rules
- Keep code clean and readable — no shortcuts that would be confusing later
- Commit often with descriptive messages so there's a clear trail of what was learned
- When something doesn't work, explain *why* it failed, not just how to fix it
- Don't build features beyond what's needed to understand the concept

# Course Site Project

A beginner course website teaching people how to set up Claude Code, Obsidian, and GitHub. Audience is complete non-technical beginners. Plain HTML/CSS only — no frameworks.

## Pages
- `index.html` — Lesson 1: Setting up Claude Code (main page / homepage)
- `obsidian.html` — Lesson 2: Downloading Obsidian
- `github.html` — Lesson 3: Creating a GitHub account

## Style conventions
- Accent color: `#d97757` (clay orange)
- Steps use numbered orange circles with custom `<ol>` styling
- Notes use an orange left-border callout box
- Each page ends with a dark (`#1a1a1a`) rounded card
- Nav bar links all three pages — active page is highlighted orange
- Copy styles manually between pages (no shared stylesheet)

## Tone rules
- Plain English, warm, short — no jargon
- No extra explanation beyond what's needed to follow the step

## Deployed
- GitHub: https://github.com/EllaEllaEllaEllaEllaElla/claude-code-guide
- Vercel: auto-deploys on push to main

## Do not
- Add frameworks or build tools
- Change the visual style without being asked
